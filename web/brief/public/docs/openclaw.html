在 OpenClaw 中使用 DDTokens

准备工作

在开始之前，请确保：

  • 已安装 OpenClaw

    • 下载并安装 OpenClaw，安装完成后可在终端中运行 openclaw 命令

  • 已获取 DDTokens API 密钥

    • 登录 DDTokens 控制台 获取您的 API 密钥（以 sk- 开头）

    提示： 如果还没有 DDTokens 账户，请先在 DDTokens 注册并获取 API 密钥。

    配置步骤

    在终端中运行以下命令启动初始化向导：

    bash 
    openclaw onboard

    按照提示依次填写以下信息：

    第一步：选择模型/认证提供商

    bash 
    Model/auth provider:

    使用方向键选择 Custom Provider，然后按回车确认。

    第二步：填写 API Base URL

    bash 
    API Base URL:

    输入以下地址：

    bash 
    https://api.ddtokens.cn/v1

    第三步：填写 API Key

    bash 
    API Key:

    粘贴您从 DDTokens 控制台 获取的 API 密钥（sk-xxxxxxxxxxxx）。

    第四步：选择端点兼容性

    bash 
    Endpoint compatibility:

    使用方向键选择 OpenAI-compatible，然后按回车确认。

    第五步：填写模型 ID

    bash 
    Model ID:

    输入您要使用的模型 ID，例如：

    bash 
    claude-sonnet-4-6

    配置完成

    完成以上步骤后，OpenClaw 将使用 DDTokens 作为后端服务。您可以直接开始使用 AI 对话功能，所有请求将通过 DDTokens 路由到对应的模型。

    常见问题

    认证失败？

    • 确认 API Key 正确且以 sk- 开头
    • 在 DDTokens 控制台 检查密钥是否有效
    • 确保账户余额充足

    模型不可用？

    • 确认 API Base URL 填写为 https://api.ddtokens.cn/v1（包含 /v1 后缀）
    • 确认 Endpoint compatibility 选择了 OpenAI-compatible
    • 检查模型 ID 是否拼写正确
